# 13 · Пакет UX-мелочей

Набор независимых, недорогих UX-улучшений. Все строки — через `t()` (группа `ux.*`, ru+en синхронно).

## Сделано

1. **Тема по системе по умолчанию.** `store.ts`: если в `localStorage` нет ключа
   `vpnhub.theme` (пользователь не выбирал тему явно), берём системную через
   `window.matchMedia('(prefers-color-scheme: dark)')`. Явный выбор всегда в приоритете
   и не ломается. Функция `initialTheme()`.

2. **React Error Boundary.** `app/App.tsx`: class `ErrorBoundaryInner` +
   функциональная обёртка `ErrorBoundary` (локализует строки через `useT()` и
   прокидывает пропами, т.к. class component не может пользоваться хуком). Оборачивает
   основной контент (`<Shell>`). При краше рендера показывает аккуратный экран
   «что-то пошло не так» + кнопка «Перезагрузить» вместо белого экрана. Ключи
   `ux.crashTitle`, `ux.crashSub`, `ux.reload`.

3. **Skeleton-загрузка.** `components/ui.tsx`: компоненты `Skeleton` (плейсхолдер с
   бегущим шиммером) и `SkeletonCard` (готовый каркас карточки). CSS `.skeleton` +
   `@keyframes shimmer` в `styles.css`. Применены вместо голого `Spinner` в списках
   Серверов (`Servers.tsx`) и Групп (`Groups.tsx`).

4. **Быстрый шаринг приглашения из списка групп.** `Groups.tsx`: у `Group` в списке
   уже есть `token`, инвайт-ссылка = `${location.origin}/join/${token}` (как в
   GroupDetail). Добавлена кнопка «Пригласить» в футере карточки — копирует ссылку в
   буфер без захода в детали. Сделана как `role="button"` внутри карточки-`<button>`
   (чтобы не вкладывать `<button>` в `<button>`), с `stopPropagation`, чтобы клик не
   открывал детали группы. Ключи `ux.shareInvite`, `ux.inviteCopied`.

## Проверки

- `make front-lint` (tsc `--noEmit`) — зелёный.
- `cd frontend && npm run build` — зелёный (пред-существующее предупреждение о размере
  чанка не в счёт).

## Осталось (отдельные будущие подзадачи)

- **Командная палитра (cmd+K)** — глобальный быстрый переход/поиск. Крупная фича,
  требует индекса действий и оверлея-палитры.
- **Bulk-операции** — множественный выбор в списках (серверы/участники) и групповые
  действия (удаление, выдача доступа). Крупная фича.
- **Сортировка списков** — по имени/статусу/дате в Серверах и Группах.
