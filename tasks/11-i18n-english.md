## 11. i18n + английский язык

**Категория** Платформа/UX · **Сложность** L · **Зависимости** нет (фундамент для UI-задач №9/№10/№14/№13)

### Зачем
Панель полностью русскоязычная, строки захардкожены в JSX. Для OSS-аудитории английский — прямой множитель охвата. Нужен механизм локализации, который не тянет тяжёлых зависимостей, проверяется на этапе сборки и позволяет наращивать перевод экран за экраном.

### Что уже есть в коде
- Самодельная плюрализация `plural(n, one, two, five)` использовалась в экранах для русского склонения.
- `store.ts` (Zustand) хранит `theme` с persist в localStorage + атрибут на `<html>` — тот же паттерн переиспользован для языка.
- `Profile.tsx` имел статичный плейсхолдер «Язык / Русский» (нерабочий) — заменён на реальный переключатель.

### Дизайн решения
**Подход: самодельный типобезопасный словарь (`frontend/src/lib/i18n.ts`), НЕ i18next.**
Почему: панель self-hosted и держит зависимости под контролем; из фич нужны только словарь с интерполяцией и русская плюрализация. Самодельный вариант даёт то, чего нет из коробки у большинства либ — проверку КЛЮЧЕЙ на этапе tsc:
- `ru` — источник истины; тип словаря `Dict` выводится из него (`keyof typeof ru` = `TKey`), поэтому `t("...")` автодополняется и опечатка = ошибка сборки;
- `en ... satisfies Dict` — tsc гарантирует, что английский покрывает РОВНО те же ключи (пропуск подсветится);
- ноль рантайма сверх пары функций, ноль веса в бандле.

**Компоненты i18n.ts:** `detectLang()` (localStorage `vpnhub.lang` → `navigator.language` → дефолт ru), `interpolate()` для `{var}`, `pickPlural()` (ru — правила склонения, en — `n===1?one:other`), `useT()` — реактивный хук (перечитывает `lang` из Zustand, т.е. смена языка мгновенно перерисовывает интерфейс), плюс `LANGS`/`LANG_LABEL`/`Lang`/`TKey`.

**Store:** `lang: Lang` + `setLang(lang)` (persist в localStorage + `document.documentElement.lang`), инициализация из `detectLang()`.

### Реализовано в этом коммите (MVP)
- Полная инфраструктура i18n (`lib/i18n.ts`) с ru+en словарями для групп `nav.*`, `common.*`, `profile.*`, `events.*`.
- Навигация (`App.tsx`): сайдбар + bottom-nav + заголовок «Администрирование» локализованы через `useT()`/`NAV_META.labelKey` — реактивно меняются при переключении языка.
- Переключатель языка RU/EN в `Profile.tsx` (рядом с переключателем темы) + локализация блока настроек (тема/язык) через `t()`.
- Определение языка при старте (navigator.language + сохранённый выбор).

### Как добавить строку/язык/экран
- Строка: добавить ключ в `ru` (источник) и тот же ключ в `en` — пропуск в `en` подсветит tsc. Использовать `const t = useT();` и `t("group.key", { var })`.
- Язык: завести объект `xx = { ... } satisfies Dict`, расширить `Lang`, `LANGS`, `LANG_LABEL`, `DICTS`.
- Экран: заменить захардкоженные строки на `t(...)`, заведя ключи по группе экрана.

### Как тестировать
tsc (`make front-lint`) — проверяет паритет ключей ru/en (через `satisfies`) и валидность ключей во всех `t(...)`; `npm run build` — сборка. Бэкенд не затронут (`make check`/`make test` остаются зелёными).

### Риски и подводные камни
- Смена языка перерисовывает только компоненты, читающие `useT()` — немигрированные экраны остаются на русском (ожидаемо для поэтапной миграции).
- Даты/телефоны форматируются отдельно (react-phone-number-input locale, backend-даты) — вне словаря.

### Критерии приёмки
- Переключатель RU/EN в профиле меняет язык навигации и блока настроек мгновенно; выбор сохраняется между сессиями; `<html lang>` обновляется.
- tsc и сборка зелёные; ключ-паритет ru/en гарантирован типами.

### Осталось (вне этого коммита)
- Миграция остальных ~13 экранов на `t(...)` (Profile/Events частично; ключи для events.* уже заведены). Делать по мере доработки экранов — задачи №9/№10/№14/№13 должны сразу использовать `t()`.
- Локализация строк, приходящих с backend (тексты ошибок, remediation title/explanation/manualSteps): механизм — коды + словарь на фронте ИЛИ локализация на бэке по заголовку `Accept-Language`.
- Полная плюрализация в мигрируемых экранах через плюрал-ключи словаря (сейчас часть экранов ещё использует локальный `plural`).
- Документация сайта остаётся русской (осознанно).
