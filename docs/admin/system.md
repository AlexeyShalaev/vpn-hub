# Система и обновления

Раздел **Система** доступен администратору. Здесь — версия продукта, обновления, состояние
инстанса и резервные копии.

Резервным копиям и мастер-ключу посвящена отдельная страница —
[Резервные копии и мастер-ключ](backups.md). Ниже — про остальное.

## Версия и обновления

Вверху показана текущая версия, образ и метка: **«актуальная версия»** или **«доступно обновление
…»**.

- **«Проверить обновления»** — запросить, есть ли новая версия. Рядом указан **канал обновлений**.
- Если обновление доступно, появится кнопка **«Обновить»**, открывающая окно релиза.

### Окно обновления

В окне обновления показано:

- **«Что нового»** — список изменений релиза.
- **«Как обновить образ»** — команда для ручного обновления на хосте:

    ```sh
    docker compose pull && docker compose up -d
    ```

- **«Обновить сейчас»** — появляется, если обновление из панели настроено на сервере: кнопка
  выполнит заранее заданную команду обновления. Если не настроено — обновите образ вручную командой
  выше.

!!! info "Данные при обновлении сохраняются"
    Обновляется образ приложения, а данные в PostgreSQL остаются на месте. Необходимые миграции базы
    применяются автоматически при старте новой версии.

## Состояние системы

Блок показывает здоровье инстанса:

- **База данных** — подключена или недоступна, движок, хост и имя БД, задержка. Если база
  недоступна, панель работать не сможет — проверьте PostgreSQL.
- **Аптайм** — сколько работает текущий процесс.
- **Последний бэкап БД** — когда была сделана последняя резервная копия.

## Об инстансе

Справочный блок: редакция, образ, дата сборки и адрес инстанса. Эти данные пригодятся при обращении
за поддержкой или диагностике.

## Предупреждение о мастер-ключе

Если мастер-ключ не задан (используется небезопасный ключ по умолчанию), вверху раздела появится
красное предупреждение. Это важно исправить — SSH-доступы к серверам иначе фактически не защищены.
Что делать — [Мастер-ключ](backups.md#master-key).
