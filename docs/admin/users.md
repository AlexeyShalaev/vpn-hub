# Пользователи

Раздел **Пользователи** доступен администратору и показывает все аккаунты инстанса. Отсюда вы
подтверждаете регистрации, редактируете данные, меняете пароли и удаляете пользователей.

## Список пользователей

Администраторы показаны выше остальных. У каждого аккаунта видно имя, метку **«Админ»** (если это
администратор), статус, телефон и дату создания. При трёх и более пользователях появляется поиск по
имени и телефону.

### Статусы аккаунта

| Статус | Значит | Может войти |
|---|---|---|
| **Активен** | Обычный рабочий аккаунт | Да |
| **В ожидании** | Зарегистрировался, но ещё не подтверждён | Нет |
| **Заблокирован** | Доступ закрыт администратором | Нет |

## Подтверждение регистраций {#approve}

Когда человек регистрируется **не по приглашению**, его аккаунт создаётся со статусом
**«В ожидании»** и войти он не может. Чтобы подтвердить:

1. Откройте пользователя в списке.
2. В поле **«Статус»** выберите **«Активен»**.
3. Нажмите **«Сохранить»**.

Чтобы временно закрыть доступ, переведите аккаунт в **«Заблокирован»**.

!!! tip "Приглашённым подтверждение не нужно"
    Если человек присоединяется по [ссылке-приглашению](../owner/groups.md#invite), его аккаунт
    активируется автоматически — подтверждать вручную не требуется.

## Редактирование пользователя {#edit}

Клик по строке открывает окно **«Данные пользователя»**:

- **Имя** и **Телефон** (телефон — это логин для входа).
- **Статус** — для обычных пользователей; администратор всегда активен, его статус не редактируется.
- **Пароль** — кнопка **«Задать новый»** позволяет вписать пароль вручную или **«Сгенерировать»**
  случайный. Старый пароль хранится в виде необратимого хэша (Argon2) и не показывается.

Сохранение применяет все изменения разом; смена пароля происходит только если вы задали новый.

## Удаление пользователя

В окне пользователя — кнопка **«Удалить пользователя»** (недоступна для администратора).
Подтвердите:

> Аккаунт будет удалён безвозвратно.

## Кто такой администратор

Администратор — это особый признак аккаунта. Администраторы видят разделы **Пользователи** и
**Система** и всегда «активны».

!!! note "Как назначаются администраторы"
    Первый администратор создаётся при [первичной настройке](../guide/first-run.md), а также
    администратора можно задать через переменные окружения при установке инстанса
    (`VPNHUB_ADMIN_PHONE` / `VPNHUB_ADMIN_PASSWORD`). В окне редактирования пользователя признак
    администратора не переключается — это делается на уровне инстанса.
